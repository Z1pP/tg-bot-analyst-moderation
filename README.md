# Telegram Bot - Аналитика и Модерация

Telegram бот для аналитики активности пользователей и модерации чатов. Предоставляет администраторам инструменты для отслеживания статистики, управления пользователями, генерации отчетов и модерации сообщений.

## 🚀 Основные возможности

### 📊 Аналитика и отчеты
- **ИИ-аналитика и сводка**: Интеллектуальный анализ логов чата за последние 24 часа с использованием нейросетей
  - **Краткая сводка**: Анализ атмосферы, главной темы и ключевых итогов обсуждения
  - **Глубокая аналитика**: Подробный разбор тональности, ключевых тем, принятых решений (Action Items), ролей участников и потенциальных рисков
  - **Контекстное управление**: Автоматическая обработка до 100 000 символов истории сообщений
  - **Удобный UX**: Сводка отправляется отдельным сообщением с кнопкой "Скрыть", не перекрывая меню управления
- **Отчеты по пользователям**: Статистика активности, рейтинги, детализация по периодам
- **Отчеты по чатам**: Анализ активности в конкретных чатах
- **Рейтинг за сутки**: Топ пользователей по активности
- **Детализация перерывов**: Анализ периодов неактивности пользователей с учетом реакций
- **Анализ рабочего времени**: Фильтрация активности по рабочим часам, расчет рабочих часов
- **Отслеживание реакций**: Учет реакций на сообщения в аналитике активности
- **Анализ ответов**: Отслеживание времени ответа на сообщения
- **Автоматическая рассылка отчетов**: Настройка индивидуального расписания ежедневной рассылки отчетов в архивные чаты с возможностью включения/отключения для каждого чата

### 👥 Управление пользователями
- Отслеживание пользователей в чатах
- Добавление/удаление пользователей из отслеживания
- Просмотр статистики по конкретному пользователю
- Управление доступом администраторов к чатам
- Управление ролями пользователей

### 🚫 Модерация
- **🛡️ Антибот-верификация**: Защита от спам-ботов через обязательную проверку новых участников:
  - Автоматический мут новых пользователей при входе в чат
  - Приветственное сообщение с кнопкой «Я человек» прямо в групповом чате
  - Кнопка срабатывает только для того пользователя, которому адресовано сообщение — нажатие от другого участника игнорируется
  - При нажатии кнопки мут снимается, сообщение удаляется
  - Если пользователь не нажал кнопку в течение 60 минут — сообщение удаляется, а пользователь кикается из чата (без постоянного бана, может вернуться)
  - **Настраиваемое приветствие**: Возможность установки кастомного текста для каждого чата с поддержкой упоминания пользователя (`@{username}`)
- **Авто-модерация (ИИ)**: Накопление текстовых сообщений в Redis; при достижении порога `AUTO_MODERATION_BATCH_SIZE` пачка отправляется в очередь **TaskIQ (RabbitMQ)**, воркер вызывает OpenRouter и при срабатывании шлёт карточку в **архивный чат** с кнопками «Заблокировать» / «Отменить» (для staff). Включение — в меню чата («Авто-модерация»); размер пачки задаётся в `.env`, не в БД.
- **Настраиваемые наказания**: Создание индивидуальной лестницы наказаний (предупреждения, муты, баны) для каждого чата
- **Интерактивное управление**: Удобный процесс настройки через инлайн-меню с автоматической очисткой чата
- **Блокировки**: Бан пользователей с указанием причины
- **Муты**: Временное ограничение отправки сообщений с гибкой настройкой длительности
- **Амнистия**: Разблокировка, размут, отмена последнего предупреждения

### 💬 Управление сообщениями
- Удаление сообщений из чатов
- Ответы на сообщения от имени бота
- Отправка сообщений в чаты
- Работа с медиа-альбомами

### 📝 Шаблоны ответов
- Создание и управление шаблонами сообщений
- Категоризация шаблонов с иерархической структурой
- Быстрый доступ к шаблонам при модерации
- Поиск шаблонов по категориям
- Статистика использования шаблонов

### 📋 Логирование
- Логирование всех действий администраторов
- История модерации
- Отчеты о действиях в архивных чатах

### 📢 Релизные заметки
- Создание и редактирование релизных заметок
- Многоязычная поддержка (ru/en)
- Рассылка релизных заметок пользователям
- Управление версиями релизных заметок

### 🌐 Многоязычность
- Автоматическое определение языка пользователя
- Поддержка русского и английского языков
- Сохранение языковых настроек пользователя
- Динамическое переключение языка интерфейса

### ⚡ Производительность
- **Единый пул Redis**: Один клиент Redis для FSM, кеша и буфера аналитики — снижение нагрузки на соединения
- **Redis кеширование**: Оптимизация запросов к БД через Redis
- **In-memory кеш для чатов с архивом**: Быстрый доступ без сериализации для корректной работы ORM
- **TTL Cache**: Кеш с временем жизни для временных данных
- **Антиспам middleware**: Защита от флуда и злоупотреблений
- **Буферизация сообщений**: Сообщения и реакции накапливаются и сохраняются в БД группами, что снижает нагрузку на базу данных и ускоряет работу бота
- **Буфер авто-модерации**: Список текстовых сообщений в Redis с атомарным сбросом при достижении N; тяжёлый вызов LLM выполняется во **воркере**, а не в обработчике апдейта

## 🛠 Технологический стек

- **Python 3.10+**
- **aiogram 3.x** - асинхронный фреймворк для Telegram Bot API
- **aiohttp** - HTTP-сервер webhook в процессе бота (`main.py`)
- **FastAPI** - отдельное приложение в `src/api/` (Mini App API); в штатном `main.py` бота **не** поднимается, при необходимости запускается своей точкой входа
- **SQLAlchemy 2.0** - ORM для работы с БД
- **PostgreSQL** - основная база данных
- **Redis** - кеширование и хранение результатов задач
- **RabbitMQ** - брокер сообщений для TaskIQ
- **Alembic** - миграции БД
- **TaskIQ** - асинхронная очередь задач и планировщик (использует RabbitMQ как брокер)
- **Docker & Docker Compose** - контейнеризация

## 📋 Требования

- Python 3.10 или выше
- PostgreSQL 14+
- Redis 7+
- RabbitMQ 3.13+
- Docker и Docker Compose (для запуска через Docker)

## 🔧 Установка

### Локальная установка (с использованием uv)

Рекомендуется использовать [uv](https://github.com/astral-sh/uv) для быстрого управления зависимостями.

1. **Клонируйте репозиторий:**
```bash
git clone <repository-url>
cd tg-bot-analyst-moderation
```

2. **Установите зависимости и создайте виртуальное окружение:**
```bash
uv sync
```

3. **Настройте переменные окружения:**
Скопируйте `env-example` в `.env` и заполните необходимые значения:
```bash
cp env-example .env
```

4. **Создайте базу данных** (если ещё не создана):
```bash
createdb dev_analytics_bot  # или используйте psql
```

5. **Примените миграции:**
```bash
cd src
alembic upgrade head
```

## 🚀 Запуск

Для полноценной работы системы необходимо запустить несколько компонентов.

### 1. Бот (aiogram)

**Режим Polling (для разработки):**
```bash
cd src
python main.py
```

**Режим Webhook (для продакшна):**
```bash
cd src
python main.py --webhook-url https://your-domain.com
```

### 2. Воркер (TaskIQ Worker)

Необходим для выполнения фоновых задач (отчёты, аналитика, **пачки авто-модерации**). Из корня проекта:
```bash
export PYTHONPATH=$PYTHONPATH:$(pwd)/src
taskiq worker scheduler.taskiq:broker tasks.report_tasks tasks.analytics_tasks tasks.moderation_tasks tasks.automoderation_tasks
```

### 3. Планировщики (Schedulers)

**TaskIQ Scheduler (рассылка отчетов):**
```bash
python src/taskiq_scheduler_entrypoint.py
```

**Analytics Scheduler (обработка буфера сообщений):**
```bash
python src/analytics_scheduler_entrypoint.py
```

### Запуск через Docker (Рекомендуется)

```bash
docker-compose up -d
```

В Docker все компоненты (бот, воркер, планировщики, БД, Redis, RabbitMQ) запускаются автоматически.

## 🏗 Архитектура

Проект построен на принципах **Clean Architecture** и разделен на функциональные слои:

- **Handlers** - точка входа для событий Telegram, обработка команд и сообщений.
- **Use Cases** - реализация конкретных бизнес-сценариев приложения.
- **Services** - вспомогательные сервисы (AI, кэширование, работа со временем).
- **Presenters** - логика форматирования данных для вывода пользователю.
- **Repositories** - абстракция над доступом к данным.
- **Models** - описание структуры базы данных (SQLAlchemy).

Для управления зависимостями используется **Dependency Injection** (библиотека `punq`). Фоновые задачи и планирование реализованы с помощью **TaskIQ**.

### Middleware
- **LanguageMiddleware** - автоматическое определение и сохранение языка пользователя
- **AlbumMiddleware** - обработка медиа-альбомов
- **AdminAntispamMiddleware** - защита от флуда для администраторов
- **AsyncErrorHandler** - централизованная обработка исключений: перехватывает необработанные ошибки в хендлерах и автоматически отправляет детальный отчёт (тип, сообщение, трейсбек) всем пользователям с ролью `DEV` в личку. Включает rate limiting (не чаще 1 раза / 60 сек на уникальную ошибку) и кеш DEV-пользователей (5 мин TTL)

### Сервисы
- **AI Summarization** - интеллектуальный анализ и сводки чатов на базе OpenRouter (GPT/Claude/Gemini); те же учётные данные используются для **авто-модерации** (батч в воркере)
- **Caching** - система кеширования через Redis с поддержкой TTL
- **AnalyticsBufferService** - буферизация сообщений и реакций в Redis для последующей батч-обработки
- **AutoModerationBufferService** - буфер текстовых сообщений для авто-модерации (Redis LIST, атомарный flush при N)
- **BreakAnalysisService** - анализ перерывов в активности с учетом реакций
- **WorkTimeService** - анализ рабочего времени, фильтрация по рабочим часам
- **TimeZoneService** - работа с часовыми поясами
- **ArchiveBindService** - управление архивными чатами
- **ReportScheduleService** - управление расписанием автоматической рассылки отчетов
- **TaskiqSchedulerService** - планировщик задач на основе TaskIQ
- **UserService** - управление пользователями с Redis-кешированием; автоматически синхронизирует `username` в БД при изменении (включая удаление username пользователем), инвалидирует устаревшие ключи кеша

```bash
alembic revision --autogenerate -m "описание изменений"
alembic upgrade head
```

## 🔔 Уведомления об ошибках для DEV

Пользователи с ролью `DEV` получают детальные уведомления об ошибках в личные сообщения:

- **Хендлеры и коллбэки** — перехватываются `AsyncErrorHandler` middleware
- **Фоновые планировщики** — `analytics_scheduler_entrypoint.py` и `taskiq_scheduler_entrypoint.py` вызывают `notify_devs_about_error()` при исключениях в рабочем цикле
- **Формат отчёта**: тип исключения, сообщение, где произошло, полный трейсбек (с обрезкой до 4096 символов)
- **Rate limiting**: одна и та же ошибка уведомляет не чаще 1 раза в 60 секунд
- **Устойчивость**: DEV-пользователи кешируются на 5 минут; при недоступности БД используются кешированные данные

## 🔐 Безопасность

- Доступ к функционалу бота только для администраторов
- Фильтрация команд по ролям пользователей
- Антиспам middleware для защиты от флуда
- Валидация входных данных
- Логирование всех действий администраторов
- Управление правами бота через BotPermissionService
- Проверка прав бота в чате через пункт меню «Проверка прав» в настройках чата
- Проверка доступа администраторов к чатам
- Защищённый пользователь (`PROTECTED_USER_TG_ID` в `.env`) — нельзя изменить роль (Telegram ID владельца/разработчика)
- Идентификация пользователей по `tg_id` (не по `username`): корректная работа с пользователями без username и синхронизация при его изменении/удалении

## 📝 Использование

### Команды бота

- `/start` - Запуск бота (только в приватных чатах)

### Основные функции

1. **Меню пользователей** (`😀 Пользователи`)
   - Добавление пользователей в отслеживание
   - Просмотр статистики по пользователям
   - Удаление из отслеживания

2. **Меню чатов** (`📝 Чаты`)
   - Добавление чатов в отслеживание
   - Получение статистики по чатам
   - Удаление чатов из отслеживания

   - В меню действий чата: **авто-модерация** (вкл/выкл); размер пачки N задаётся только в `AUTO_MODERATION_BATCH_SIZE` (`.env`), не в интерфейсе
3. **Отчеты** (`⏱️ Получить отчет`)
   - ИИ-аналитика за 24 часа (краткая или полная)
   - Отчеты по конкретному чату
   - Отчеты по пользователям
   - Рейтинг за сутки
   - Выбор периода (день, неделя, месяц, кастомный)

4. **Модерация** (`🚫 Блокировки`)
   - Предупреждения пользователей
   - Блокировки с указанием причины
   - Амнистия (разблок, размут, отмена предупреждения)
   - **🛡️ Антибот** (в меню действий чата):
     - Включение/выключение обязательной проверки новых участников
     - При входе новый участник получает мут и сообщение с кнопкой «Я человек» в чате
     - Кнопка работает только для того пользователя, которому адресована
     - При нажатии — мут снимается, сообщение удаляется
     - При бездействии в течение 5 минут — пользователь кикается, сообщение удаляется
   - **👋 Приветствие** (в меню действий чата):
     - Установка кастомного текста приветствия для новых участников
     - Поддержка тега `{username}` для обращения к пользователю
   - **Настройка наказаний** (в меню действий чата ⚖️):
     - Создание новой последовательности наказаний
     - Настройка длительности мутов (10m, 2h, 1d и т.д.)
     - Сброс к настройкам по умолчанию
   - **🔍 Проверка прав** (в меню действий чата): Детальная проверка прав бота в чате через Telegram API — статус, права администратора (удаление, мут/бан, приглашения и т.д.) с пояснениями, что может или не может бот при наличии/отсутствии каждого права

5. **Управление сообщениями** (`💬 Упр. сообщенями`)
   - Удаление сообщений
   - Ответы на сообщения
   - Отправка сообщений в чаты

6. **Шаблоны** (`🔖 Шаблоны`)
   - Создание шаблонов ответов
   - Управление категориями
   - Использование шаблонов при модерации
   - Поиск шаблонов по категориям

7. **Релизные заметки** (`📝 Релизные заметки`)
   - Создание и редактирование релизных заметок
   - Многоязычная поддержка
   - Рассылка релизных заметок пользователям
   - Управление версиями

8. **Архивные чаты**
   - Привязка архивных чатов
   - Уведомления в архив: новый участник, выход участника, кик непроверенного (антибот), карточки авто-модерации (при включённой функции и привязанном архиве)
   - Получение статистики по архивным чатам
   - Отчеты о действиях в архивных чатах
   - Настройка автоматической ежедневной рассылки отчетов (время отправки, включение/отключение)
   - Управление расписанием отправки отчетов (включение/отключение, выбор времени)

## 🐛 Решение проблем

### Бот не отвечает

1. Проверьте токен бота в `.env`
2. Убедитесь, что бот запущен: `docker-compose ps`
3. Проверьте логи: `docker-compose logs bot`

### Ошибки подключения к БД

1. Проверьте настройки БД в `.env`
2. Убедитесь, что PostgreSQL запущен
3. Проверьте доступность БД: `psql -h localhost -U postgres`

### Проблемы с миграциями

1. Проверьте URL БД для Alembic в `.env`
2. Убедитесь, что используется синхронный драйвер (`postgresql://` вместо `postgresql+asyncpg://`)
3. Попробуйте применить миграции вручную: `alembic upgrade head`

### Проблемы с Redis

1. Проверьте доступность Redis: `redis-cli ping`
2. Убедитесь, что Redis запущен: `docker-compose ps redis`
3. Проверьте настройки Redis в `.env`: `REDIS_HOST`, `REDIS_PORT`, `REDIS_DB` (для локального запуска: host `localhost`, port `6379`)
4. **`ReadOnlyError`**: если TaskIQ бросает `You can't write against a read only replica` — `REDIS_HOST` указывает на реплику. Смените значение на адрес мастер-ноды Redis

### Проблемы с RabbitMQ

1. Проверьте доступность RabbitMQ: `docker-compose ps rabbitmq`
2. Проверьте логи RabbitMQ: `docker-compose logs rabbitmq`
3. Убедитесь, что настройки в `.env` совпадают с параметрами в `docker-compose.yml` (`RABBITMQ_USER`, `RABBITMQ_PASS`, `RABBITMQ_HOST`, `RABBITMQ_PORT`)
4. Доступ к панели управления (Management UI): `http://localhost:15672` (по умолчанию `guest`/`guest`)

### DEV-пользователи не получают уведомления об ошибках

1. Убедитесь, что в БД есть пользователи с ролью `DEV`
2. Проверьте, что у них задан `tg_id`
3. Rate limiting: одна ошибка уведомляет не чаще 1 раза в 60 секунд — это нормально
4. Ошибки из фоновых планировщиков покрываются только если они запущены (`analytics_scheduler_entrypoint.py`, `taskiq_scheduler_entrypoint.py`); ошибки внутри Taskiq-задач логируются в stdout воркера

### Проблемы с Cloudflare Tunnel (Docker)

1. Проверьте логи туннеля: `docker-compose logs tunnel`
2. Убедитесь, что туннель получил публичный URL: `curl http://localhost:4040/metrics`
3. Проверьте, что бот дождался URL туннеля перед запуском (см. docker-compose.yml)
4. Ошибка Telegram `bad webhook: Failed to resolve host` сразу после старта: публичный DNS для `*.trycloudflare.com` может обновиться с задержкой; в `main.py` предусмотрены повторные попытки `setWebhook`. При смене поддомена туннеля обновите URL или перезапустите стек

### Авто-модерация не срабатывает

1. В настройках чата должны быть включены авто-модерация и привязан архив (для карточки в архиве).
2. Должен работать **worker** с модулем `tasks.automoderation_tasks` (см. команду `taskiq worker` в README и `docker-compose.yml`).
3. Проверьте `OPEN_ROUTER_*` и логи воркера; порог N — `AUTO_MODERATION_BATCH_SIZE` в `.env`.
